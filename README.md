# NCRC members map

Static Mapbox GL map with a weekly BigQuery → Mapbox tileset job on Google Cloud Run, plus an optional BigQuery-backed **state counts** API for choropleth totals.

## Prerequisites

- **Google Cloud**: project, billing, APIs (Cloud Run, Cloud Build, BigQuery, Artifact Registry, Secret Manager). A service account with the roles from `scripts/setup-service-account.sh`.
- **Mapbox**: account, upload token (job) and public token (web). Restrict public token URLs to your deployed origin.
- **Local tools**: `gcloud`, `docker` (for image builds via Cloud Build), `python3`.

## Local development

1. Copy `.env.example` to `.env` and fill in Mapbox URLs, tilesets, `BQ_TABLE`, etc.
2. Run `./scripts/dev-local.sh` — generates `js/config.local.js` from `.env` and serves on `http://127.0.0.1:8765/`.
3. Before committing, if `config.local.js` was overwritten with real values, restore the committed stub:

   ```bash
   git checkout -- js/config.local.js
   ```

   Or leave only empty strings in `js/config.local.js` (never commit Mapbox tokens in this file).

## What not to commit

- `.env` (gitignored) — real project IDs and tokens.
- `member-map-sa-key.json` or `*-sa-key.json` (gitignored).
- `data/state_counts.json` is gitignored by default (generated by the job).

## Deploy to Cloud Run

1. Ensure **Secret Manager** has real values for the names used in `scripts/deploy-cloud-run.sh` (defaults: `mapbox-members-upload-token`, `mapbox-members-public-token`). The first deploy may create placeholders — replace them and redeploy.
2. From the repo root, with `.env` loaded (script uses `scripts/lib/common.sh`):

   ```bash
   ./scripts/deploy-cloud-run.sh
   ```

   This builds and pushes the **web**, **export job**, and **counts** images, updates the job, deploys **ncrc-members-counts** (BigQuery → JSON with `STATE_COUNTS_CACHE_TTL_SEC` from `.env`), then the **web** service.

3. Note the printed **state counts API** URL. Set `MEMBERS_STATE_COUNTS_URL` in `.env` to that base URL (with trailing `/` or your path), then run the deploy script again so the **web** service gets the new env (or update Cloud Run env for the web service in the console).

4. Smoke-test: open the web Cloud Run URL, confirm the map loads and state totals look correct.

## GitHub Actions (weekly tileset export)

The workflow `.github/workflows/weekly-members-export.yml` runs the Cloud Run job on a schedule. Configure repository **Secrets**:

| Secret | Purpose |
|--------|---------|
| `WIF_PROVIDER` | Workload Identity Federation provider resource name |
| `WIF_SERVICE_ACCOUNT` | Service account email (same as `MEMBER_MAP_SA_EMAIL`) |
| `GCP_PROJECT_ID` | GCP project ID |
| `GCP_REGION` | Region (e.g. `us-east1`) |
| `GCP_RUN_JOB_NAME` | Cloud Run job name (e.g. `ncrc-members-export`) |

Use `scripts/setup-github-oidc.sh` / `scripts/setup-github-secrets.sh` as needed. OIDC avoids storing GCP JSON keys in GitHub.

## If `js/config.local.js` ever held real tokens

That file is safe to ship as the empty stub above. If a token was previously committed, **rotate** the Mapbox token in the Mapbox dashboard and update Secret Manager / `.env`.
